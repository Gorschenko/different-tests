По структуре:

- B-tree — стандартный (для =, >, <, BETWEEN).
- Hash — только для точного равенства (=).
- GiST / SP-GiST — для геоданных, полнотекстового поиска, диапазонов.
- GIN — для массивов, JSON, полнотекста (быстрый поиск внутри составных данных).
- BRIN — для очень больших таблиц, где данные физически отсортированы (хранит краткие сводки по блокам).

По функциональности:

- Обычный (Single-Column) — по одному полю.
- Составной (Composite) — по нескольким полям (важен порядок столбцов).
- Уникальный (Unique) — запрещает дубликаты.
- Первичный ключ (PRIMARY KEY) — уникальный + NOT NULL.
- Функциональный (Expression-based) — по результату функции (LOWER(name)).
- Покрывающий (Covering / INCLUDE) — включает дополнительные поля в листья, чтобы избежать чтения таблицы.
- Частичный (Partial / Filtered) — только для подмножества строк (WHERE status = 'active').
- Кластерный (Clustered) — определяет физический порядок строк в таблице (в PostgreSQL — аналог CLUSTER, в MySQL InnoDB — сама таблица).
- Некластерный (Non-Clustered) — отдельная структура, ссылающаяся на строки.
