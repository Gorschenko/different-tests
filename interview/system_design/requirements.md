1. Функциональные требования отвечают на вопрос "Что делает система?".

1.1. Расписать функции системы.
1.2. Уточнить самые важные функции системы.
1.3. Расписать действия пользователя (поможет с проектирование АПИ).
1.4. Уточнить действия пользователей.

2. Нефункциональные требования отвечают на вопрос "Как система работает". Это критерии качества.

2.1. Performance. Производительность.
2.2. Availability. Доступность. Время непрерывной работы.
2.2. Reliability. Надежность. Процент ответов без ошибок.
2.3. Consistency. Согласованность данных.
2.4. Latency. Задержка. Время ответа.

Производительность:

1. Уточнить кол-во пользователей.
2. Как часто будут запросы?. На основе действий пользователей (см. п. 1.3) расписать их активность.
3. Выяснить пиковые часы системы.

- MAU (mounthly active users)
- DAU (daily active users) = MAU x вовлеченность %.
- RPS = Кол-во запросов пользователя в сутки x DAU / активные часы. Расчет для одного действия.
- 1 сутки = 86400 секунд.

Пример:

1. MAU = 1_000_000.
2. DAU = 500_000
3. Активные часы, например, 8 часов или 28800 секунд.
4. Считаем кол-во RPS по каждому действию пользователя. Например, пользователь смотрит ленту 20 раз в день, тогда RPS = 20 х 500_000 / 28800 = 360.
5. Разбиваем действия на чтение и запись. Далее считаем общий RPS на чтение и запись.

Если нужно посчитать размер данных: RPS x размер единицы данных x кол-во секунд в году.
